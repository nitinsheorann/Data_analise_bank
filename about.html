Core Capabilities

The system is built around the essential workflows of a retail banking platform: customer onboarding, secure authentication, and safe fund transfers.

Customer Onboarding

Create accounts with name, phone number, ATM number, and password. Input validation and uniqueness checks ensure clean, consistent data in the bankdata table.

ATM-Based Login

Customers sign in using ATM number and password. Flask session management keeps users logged in securely while protecting sensitive operations behind server-side checks.

Fund Transfer Engine

Transfers validate target accounts, check balance, and then perform atomic debit/credit updates using SQL transactions – minimizing the risk of inconsistent states.

Architecture Overview

The Bank Management System follows a simple but powerful 3-layer structure: Presentation, Application, and Data. This mirrors how many real-world fintech systems are designed.

Presentation Layer

  • HTML & CSS based UI with dedicated pages for Login, Home, Transfers, and Account Creation.
  • Clear call-to-actions and message areas for success/error feedback.
  • Responsive design that adapts to desktop and mobile layouts.

Application Layer (Flask)

  • Flask routes handle login, signup, home dashboard and money transfers.
  • Session stores the logged-in customer's ATM number securely.
  • Business rules implemented in Python: validation, balance checks, and error handling.

Data Layer (PostgreSQL)

  • PostgreSQL database hosted on Neon for cloud-based storage.
  • psycopg2 used with parameterized queries to prevent SQL injection.
  • Transactional updates ensure money movements are consistent and durable.

Security, Reliability & Tech Stack

Even as a demo / academic project, the platform is designed with real security and reliability patterns in mind.

Security & Reliability

  • Session-based authentication for every sensitive action.
  • Server-side validation of ATM number, password and transfer amounts.
  • Parameterized SQL queries throughout to avoid injection attacks.
  • Balance checks and transactional updates to prevent negative balances.
  • Clear, controlled error messages instead of exposing low-level errors.
